# BCF Chatbox – Client Review & Approval Checklist

For BCF Confection to review and sign off before launch. Each item should be
ticked only once the client has explicitly approved it.

## 1. Content approval

- [ ] Approved Dutch answers are correct
- [ ] Minimum order quantity wording is approved
- [ ] Delivery time wording is approved
- [ ] Production methods (CMT / RMG) wording is approved
- [ ] Product range wording is approved
- [ ] Pricing / offerte wording is approved
- [ ] Contact details are correct (email, phone, address)
- [ ] Any pending-verification topics remain disabled until approved

> Pending-verification topics currently disabled (not shown to visitors until
> approved): sustainability/certification, baby & children textile,
> certificates, production location.

## 2. Behavior approval

- [ ] The chatbox only answers from approved content
- [ ] Unsupported questions trigger the fallback message
- [ ] The fallback asks permission before forwarding the question
- [ ] The chatbox does not provide exact prices
- [ ] The chatbox does not make delivery promises beyond approved wording
- [ ] The chatbox does not pretend to be a human

## 3. Email handoff approval

- [ ] User consent is required before collecting contact details
- [ ] Final confirmation appears before sending
- [ ] Email recipient for production is approved
- [ ] Sender / from address for production is approved
- [ ] Client understands when production emails will begin

## 4. Privacy approval

- [ ] Conversation is stored only during the browser session
- [ ] No full conversation history is sent in the MVP
- [ ] Personal data is collected only for email handoff
- [ ] Privacy wording inside the chatbox is approved

## 5. Launch approval

- [ ] Dutch MVP can be launched first
- [ ] French version is planned for later
- [ ] Client approves the first production test email
- [ ] Client approves enabling production email routing
